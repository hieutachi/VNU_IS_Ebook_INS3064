CHAR vs VARCHAR

CHAR(N)

  • Fixed length
  • Always uses N bytes
  • Faster for fixed data
  • Use for: codes, status

VARCHAR(N)

  • Variable length
  • Uses only needed bytes
  • More flexible
  • Use for: names, descriptions

ğŸ”’ Constraints

Constraints enforce rules on data to ensure integrity:

CREATE TABLE products ( -- PRIMARY KEY: Primary key, unique, not null id INT AUTO_INCREMENT PRIMARY KEY, -- NOT NULL: Must have value name VARCHAR(100) NOT NULL, -- UNIQUE: Unique value sku VARCHAR(50) UNIQUE, -- DEFAULT: Default value status VARCHAR(20) DEFAULT 'active', -- CHECK: Check condition (MySQL 8.0+) price DECIMAL(10,2) CHECK (price > 0) );

Common Constraints

  • PRIMARY KEY - Unique identifier
  • NOT NULL - Required field
  • UNIQUE - No duplicates
  • DEFAULT - Default value
  • CHECK - Validate condition

ğŸ”¢ AUTO_INCREMENT

Automatically generates unique ID for each new row:

-- Auto increment ID CREATE TABLE users ( id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(100) ); -- Insert without ID INSERT INTO users (name) VALUES ('John'); -- id = 1 (automatic) INSERT INTO users (name) VALUES ('Jane'); -- id = 2 (automatic)

How It Works

  • Starts from 1 (or specified value)
  • Increments by 1 for each new row
  • Never reuses deleted IDs
  • Only one AUTO_INCREMENT column per table

ğŸ’¡ Best Practice: Always use AUTO_INCREMENT for primary keys - it's fast and reliable!

ğŸ“ Example: Users Table
