üì§ GET Method

Form with GET

<form action="process.php" method="GET"> <input type="text" name="search"> <button type="submit">Search</button> </form> <!-- URL: process.php?search=php -->

Processing GET Data

<?php // process.php $search = $_GET['search'] ?? ''; echo "You searched for: $search"; ?>

üí° GET Characteristics:

  • Data visible in URL
  • Can be bookmarked
  • Limited length (~2048 chars)
  • Use for search, filters

üì§ POST Method

Form with POST

<form action="process.php" method="POST"> <input type="text" name="username"> <input type="password" name="password"> <button type="submit">Login</button> </form>

Processing POST Data

<?php // process.php $username = $_POST['username'] ?? ''; $password = $_POST['password'] ?? ''; echo "Username: $username"; ?>

üí° POST Characteristics:

  • Data hidden in body
  • More secure
  • Unlimited length
  • Use for login, registration

‚öñÔ∏è Comparison: GET vs POST

Feature GET POST
Data Visibility Visible in URL Hidden in body
Length Limit Limited (~2048 chars) Unlimited
Security Poor (exposed) Better
Bookmark Can bookmark Cannot bookmark
Cache Can cache Cannot cache
Usage Search, filters Login, registration

‚ö†Ô∏è Important: Always use POST for sensitive data like passwords!

üåê Superglobals

PHP provides special arrays called superglobals that are available in all scopes:

<?php // $_GET - Data from URL $id = $_GET['id']; // $_POST - Data from POST form $name = $_POST['name']; // $_REQUEST - Both GET and POST $data = $_REQUEST['data']; // $_SERVER - Server information $method = $_SERVER['REQUEST_METHOD']; // $_FILES - File upload $file = $_FILES['avatar']; ?>

Common Superglobals

Variable Purpose
$_GET GET request data
$_POST POST request data
$_REQUEST GET, POST, and COOKIE data
$_SERVER Server and environment info
$_FILES Uploaded files

‚úÖ Check Form Submission

Method 1: Check REQUEST_METHOD

<?php if ($_SERVER['REQUEST_METHOD'] === 'POST') {{ // Process form }} ?>

Method 2: Check isset

<?php if (isset($_POST['submit'])) {{ // Process form }} ?>

Method 3: Check empty

<?php if (!empty($_POST['username'])) {{ // Has data }} ?>

üí° Best Practice: Use $_SERVER['REQUEST_METHOD'] for most reliable check.

‚úÖ Data Validation

Validation Functions

<?php $input = $_POST['input'] ?? ''; // Check empty if (empty($input)) {{ echo "Cannot be empty"; }} // Check length if (strlen($input) < 3) {{ echo "Minimum 3 characters"; }} // Check if numeric if (!is_numeric($input)) {{ echo "Must be a number"; }} // Validate email if (!filter_var($input, FILTER_VALIDATE_EMAIL)) {{ echo "Invalid email format"; }} // Validate URL if (!filter_var($input, FILTER_VALIDATE_URL)) {{ echo "Invalid URL"; }} // Regex validation if (!preg_match("/^[a-zA-Z]+$/", $input)) {{ echo "Only letters allowed"; }} ?>

üßπ Data Sanitization

Clean and secure user input before processing:

<?php $input = $_POST['input'] ?? ''; // Remove whitespace from start/end $input = trim($input); // Remove backslashes $input = stripslashes($input); // Convert special characters to HTML entities $input = htmlspecialchars($input, ENT_QUOTES, 'UTF-8'); // Filter sanitize $email = filter_var($_POST['email'], FILTER_SANITIZE_EMAIL); $url = filter_var($_POST['url'], FILTER_SANITIZE_URL); $int = filter_var($_POST['number'], FILTER_SANITIZE_NUMBER_INT); ?>

‚ö†Ô∏è Security: Always sanitize user input to prevent XSS attacks!

‚öôÔ∏è Comprehensive Validation Functions

<?php function sanitize($data) {{ $data = trim($data); $data = stripslashes($data); $data = htmlspecialchars($data, ENT_QUOTES, 'UTF-8'); return $data; }} function validateEmail($email) {{ return filter_var($email, FILTER_VALIDATE_EMAIL); }} function validateLength($str, $min, $max) {{ $len = strlen($str); return $len >= $min && $len <= $max; }} function validatePassword($password) {{ // At least 8 chars, uppercase, lowercase, number return preg_match('/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d]{{8,}}$/', $password); }} ?>

üí° Reusable Functions: Create validation functions once, use everywhere!
