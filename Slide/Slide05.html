üìñ What is SQL?

SQL = Structured Query Language

The language used to communicate with databases.

üéí Real-life Analogy

SQL is like the language to talk to a librarian:

  • "Show me all books" ‚Üí SELECT * FROM books
  • "Find books by author X" ‚Üí SELECT * FROM books WHERE author = 'X'
  • "Add a new book" ‚Üí INSERT INTO books...

Types of SQL Commands

Type Description Commands
DDL Data Definition CREATE, ALTER, DROP
DML Data Manipulation SELECT, INSERT, UPDATE, DELETE
DCL Data Control GRANT, REVOKE

üìñ SELECT - Query Data

Basic Syntax

-- Get all columns SELECT * FROM students; -- Get specific columns SELECT name, email FROM students; -- Set alias for column SELECT name AS 'Full Name', email AS 'Email' FROM students;

SELECT Structure

Syntax: SELECT columns FROM table_name;

  • * = All columns
  • Column names = Specific columns
  • AS = Alias (rename column in result)

üîç WHERE - Filter Data

Comparison Operators

-- Equal SELECT * FROM students WHERE age = 20; -- Not equal SELECT * FROM students WHERE age != 20; SELECT * FROM students WHERE age <> 20; -- Greater/Less than SELECT * FROM students WHERE age > 18; SELECT * FROM students WHERE age >= 18; SELECT * FROM students WHERE age < 25; SELECT * FROM students WHERE age <= 25;

Special Operators

-- BETWEEN - In range SELECT * FROM students WHERE age BETWEEN 18 AND 25; -- IN - In list SELECT * FROM students WHERE class IN ('CS1', 'CS2', 'CS3'); -- LIKE - Pattern search SELECT * FROM students WHERE name LIKE 'John%'; -- Starts with SELECT * FROM students WHERE name LIKE '%Smith'; -- Ends with SELECT * FROM students WHERE name LIKE '%John%'; -- Contains -- IS NULL / IS NOT NULL SELECT * FROM students WHERE phone IS NULL; SELECT * FROM students WHERE phone IS NOT NULL;

üîÄ Logical Operators: AND, OR, NOT

AND - Both Conditions

SELECT * FROM students WHERE age >= 18 AND gpa >= 3.0;

OR - At Least One

SELECT * FROM students WHERE class = 'CS1' OR class = 'CS2';

NOT - Negation

SELECT * FROM students WHERE NOT class = 'CS1';

Combining Operators

SELECT * FROM students WHERE (class = 'CS1' OR class = 'CS2') AND gpa >= 3.0;

üí° Tip: Use parentheses () to group conditions and control order of evaluation!

üìä ORDER BY - Sort Data

Sort Ascending (Default)

-- Sort by name (A-Z) SELECT * FROM students ORDER BY name; SELECT * FROM students ORDER BY name ASC;

Sort Descending

-- Sort by GPA (highest first) SELECT * FROM students ORDER BY gpa DESC;

Sort Multiple Columns

-- Sort by class first, then GPA SELECT * FROM students ORDER BY class ASC, gpa DESC;

üí° Use Cases: Display top students, newest posts, alphabetical lists, etc.

üìÑ LIMIT - Limit Results

Get First N Records

-- Get first 10 records SELECT * FROM students LIMIT 10;

Pagination

-- Page 1 (records 1-10) SELECT * FROM students LIMIT 0, 10; -- Page 2 (records 11-20) SELECT * FROM students LIMIT 10, 10; -- Page 3 (records 21-30) SELECT * FROM students LIMIT 20, 10;

Formula

Pagination Formula: LIMIT (page-1)*pageSize, pageSize

  • Page 1: LIMIT 0, 10
  • Page 2: LIMIT 10, 10
  • Page 3: LIMIT 20, 10

üîç DISTINCT - Remove Duplicates

-- Get unique classes SELECT DISTINCT class FROM students; -- Count unique classes SELECT COUNT(DISTINCT class) FROM students;

Example

Without DISTINCT With DISTINCT
CS1, CS1, CS2, CS2, CS3 CS1, CS2, CS3

üí° Use Case: Get list of unique values, count distinct items, etc.

‚ûï INSERT - Add Data

Insert Single Record

-- Insert with all columns INSERT INTO students (name, email, age, class, gpa) VALUES ('John Doe', 'john@email.com', 20, 'CS1', 3.5); -- Insert with NULL values INSERT INTO students (name, email, age, class, gpa) VALUES ('Jane Smith', 'jane@email.com', NULL, 'CS1', NULL);

Insert Multiple Records

INSERT INTO students (name, email, age, class, gpa) VALUES ('John Doe', 'john@email.com', 20, 'CS1', 3.5), ('Jane Smith', 'jane@email.com', 21, 'CS2', 3.8), ('Bob Johnson', 'bob@email.com', 19, 'CS1', 3.2);

üí° Best Practice: Always specify column names for clarity and safety!

‚úèÔ∏è UPDATE - Modify Data

Update Single Column

UPDATE students SET gpa = 3.6 WHERE id = 1;

Update Multiple Columns

UPDATE students SET gpa = 3.6, class = 'CS2' WHERE id = 1;

Update with Complex Condition

UPDATE students SET is_active = FALSE WHERE gpa < 2.0 AND age > 25;

‚ö†Ô∏è DANGEROUS: UPDATE students SET gpa = 0;

This updates ALL records! Always use WHERE clause!

üóëÔ∏è DELETE - Remove Data

Delete Single Record

DELETE FROM students WHERE id = 1;

Delete Multiple Records

DELETE FROM students WHERE class = 'CS1' AND gpa < 2.0;

TRUNCATE vs DELETE

DELETE

  • Can use WHERE
  • Doesn't reset AUTO_INCREMENT
  • Can be rolled back

TRUNCATE

  • Deletes all records
  • Resets AUTO_INCREMENT
  • Faster

‚ö†Ô∏è DANGEROUS: DELETE FROM students;

This deletes ALL records! Always use WHERE clause!

üìä Aggregate Functions

Basic Functions

-- COUNT - Count records SELECT COUNT(*) FROM students; SELECT COUNT(*) FROM students WHERE class = 'CS1'; -- SUM - Sum values SELECT SUM(gpa) FROM students; -- AVG - Average SELECT AVG(gpa) FROM students; SELECT AVG(gpa) FROM students WHERE class = 'CS1'; -- MAX / MIN SELECT MAX(gpa) FROM students; SELECT MIN(gpa) FROM students;

Combining Functions

SELECT COUNT(*) AS total_students, AVG(gpa) AS average_gpa, MAX(gpa) AS highest_gpa, MIN(gpa) AS lowest_gpa FROM students;

üìù String Functions

-- CONCAT - Concatenate strings SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM users; -- UPPER / LOWER SELECT UPPER(name) FROM students; SELECT LOWER(email) FROM students; -- LENGTH SELECT name, LENGTH(name) AS name_length FROM students; -- SUBSTRING SELECT SUBSTRING(name, 1, 5) FROM students; -- TRIM SELECT TRIM(name) FROM students; -- REPLACE SELECT REPLACE(email, '@gmail.com', '@yahoo.com') FROM students;

Common Use Cases

  • CONCAT - Combine first and last name
  • UPPER/LOWER - Normalize text
  • LENGTH - Validate input length
  • SUBSTRING - Extract part of string

üìÖ Date Functions

-- NOW() - Current date and time SELECT NOW(); -- CURDATE() - Current date SELECT CURDATE(); -- DATE_FORMAT SELECT DATE_FORMAT(created_at, '%d/%m/%Y') FROM students; SELECT DATE_FORMAT(created_at, '%H:%i:%s') FROM students; -- DATE_ADD / DATE_SUB SELECT DATE_ADD(NOW(), INTERVAL 7 DAY); SELECT DATE_SUB(NOW(), INTERVAL 1 MONTH); -- DATEDIFF - Days between dates SELECT DATEDIFF(NOW(), created_at) AS days_ago FROM students; -- YEAR, MONTH, DAY SELECT YEAR(created_at), MONTH(created_at), DAY(created_at) FROM students;

üîÄ Conditional Functions

IF Function

SELECT name, IF(gpa >= 3.0, 'Good', 'Average') AS level FROM students;

CASE WHEN

SELECT name, gpa, CASE WHEN gpa >= 3.6 THEN 'Excellent' WHEN gpa >= 3.2 THEN 'Good' WHEN gpa >= 2.5 THEN 'Fair' WHEN gpa >= 2.0 THEN 'Average' ELSE 'Poor' END AS level FROM students;

COALESCE

-- First non-NULL value SELECT COALESCE(phone, email, 'N/A') AS contact FROM students;

üìù Example: Student Management

Common Queries

-- 1. Get all students SELECT * FROM students; -- 2. Get students in class CS1 SELECT * FROM students WHERE class = 'CS1'; -- 3. Get students with GPA >= 3.5 SELECT name, gpa FROM students WHERE gpa >= 3.5; -- 4. Sort by GPA descending SELECT * FROM students ORDER BY gpa DESC; -- 5. Top 3 students SELECT * FROM students ORDER BY gpa DESC LIMIT 3;

üìä Example: Statistics by Class

-- Statistics by class SELECT class, COUNT(*) AS total, AVG(gpa) AS avg_gpa, MAX(gpa) AS max_gpa, MIN(gpa) AS min_gpa FROM students GROUP BY class;

Other Useful Queries

-- Students without phone SELECT * FROM students WHERE phone IS NULL; -- Search by name SELECT * FROM students WHERE name LIKE '%John%'; -- Update GPA UPDATE students SET gpa = 3.6 WHERE id = 1; -- Delete student DELETE FROM students WHERE id = 5;
